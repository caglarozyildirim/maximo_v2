2. BULUNAN ÖNEMLİ BİLGİLER
================================================================================

2.1 TEMEL İSTATİSTİKLER
  - Toplam Form Alanı: 70+ alan
  - Zorunlu Alanlar: 45+
  - Opsiyonel Alanlar: 25+
  - Onay Seviyeleri: 4 (Maliyet Merkezi, Teknik Şef, Bakım Müdürü, Muhasebe)
  - Ekran Sayısı: 4 ekran (Oluşturma, Liste, Yazdırma, Rapor)

2.2 TEMEL VARLıK BİLGİSİ (MANDATORY)
  - SAP Varlık Numarası (Text) - SAP'ten otomatik
  - SAP Varlık Adı (Text) - SAP'ten otomatik
  - Bakım Varlık ID (Text/Number) - Sistem tarafından otomatik
  - Bakım Varlık Adı (Text) - SAP'ten otomatik
  - Üretici Adı, Modeli, Seri Numarası

2.3 MALİ BİLGİLER (MANDATORY - READ ONLY)
  - IFRS Amortisman Anahtarı, Faydalı Ömrü, Aktifleştirme Tarihi
  - IFRS Para Birimi, APC, Birikmiş Amortisman, Cari Defter Değeri
  - Yerel Standart: Amortisman, Faydalı Ömrü, Para Birimi, APC, vb.
  - Hepsi SAP'ten otomatik doldurulur, düzenlenmez

2.4 HURDA YÖNTEMİ BİLGİSİ (MANDATORY)
  - Hurda Yöntemi Dropdown: Selling, Scrapping, Donation, Write-off
  - Selling seçilirse:
    - Satış Fiyatı (Currency) - MANDATORY
    - Para Birimi (EUR, USD, TRY) - MANDATORY
    - Fatura İsteme Formu (Checkbox) - MANDATORY

2.5 ONAY SÜRECİ (MANDATORY)
  Adım 1: İstek Oluştur
  Adım 2: Maliyet Merkezi Sorumlusu Onayı
    - Approval from cost center responsible (Checkbox)
    - Approval datetime (Auto)
    - Approver Info (User Lookup)
  
  Adım 3: Teknik Şef (SL-Engineer) Onayı (Koşullu: bakım varlık ID varsa)
    - Approval from SL-Engineer (Checkbox)
    - Approval datetime (Auto)
    - Approver Info (User Lookup)
  
  Adım 4: Bakım Müdürü Onayı
    - Approval of Maintenance manager (Checkbox)
    - Approval datetime (Auto)
    - Approver Info (User Lookup)
  
  Adım 5: Mali/Muhasebe Onayı
    - Asset Retirement Accounting Approval (Checkbox)
    - Approval Options: Approve, Reject, Reevaluate
    - Reevaluate seçilirse:
      - Reevaluation date (Date) - MANDATORY
      - Reevaluation Explanation (Textarea) - MANDATORY
  
  Adım 6: Fiziksel Hurda Onayı
    - Physical Retirement approval (Checkbox)
    - Status: "Not yet Required" (Gelecekte aktif olacak)

2.6 KULLANıCI BİLGİ ALANLAR
  - Request created by user info (Otomatik - Oturum açan)
  - Requestor user info (Manuel/Otomatik)
  - Cost Center Asset Responsible info (SAP'ten otomatik)
  - Approvers at each step (Manuel seçim)
  - Format: **[ID] - [Name] - [Department]
  - Örnek: **23 - Yasin Taşdelen - FIPC

2.7 BELGELENDİRME
  - Documents Grid (Attachment Table)
  - Sütunlar: ID, Type, Title, Creation Date time
  - Add Document Button: PDF, DOCX, XLSX, JPG, PNG, vb. destekler

2.8 TARİH/ZAMAN ALANLAR
  - SAP Veri Alış Tarihi (Auto - Read-only)
  - İstek Oluşturma Tarihi (Auto - Read-only)
  - Onay Tarihleri (Auto sırasında)
  - Yeniden Değerlendirme Tarihi (Manuel - Koşullu)

2.9 İŞLEM BİLGİSİ
  - İstek ID: Auto-generated
  - Durum: Waiting, In Progress, Approved, Rejected, Completed
  - Hurda Nedeni (Reason): Textarea - Zorunlu

2.10 BUSINESS KURALLAR
  - SAP Entegrasyonu: Varlık ve mali bilgiler otomatik SAP'ten
  - Bakım Varlık Desteği: Bakım varlığı olmayan varlıklar da işlenebilir
  - IFRS ve Yerel Standart Uyumluluğu
  - 4 seviye zorunlu onay (Cost Center > Engineer > Manager > Accounting)
  - Koşullu görünürlük (Selling seçilince satış fiyatı görül, vb.)

2.11 KOŞULLU GÖRÜNÜRLÜK (Conditional Visibility)
  - Satış Fiyatı + Fatura Formu: IF Retirement Method = "Selling"
  - Teknik Şef Onayı: IF Maintenance asset ID EXISTS
  - Reevaluation Alanları: IF Accounting Approval = "Reevaluate"
